# response_feedback.py — 句间结构识别层建成

**日期**: 2026-07-15 12:00 GMT+8
**状态**: ✅ 64/64 自测全过，6/6 集成全过

## 背景

用户指出：鸡鸣寺樱花例子中，意义不在词里——"鸡鸣寺的樱花开了" + "你要和我一起去看看吗？"——两句合在一起的意思是"因为是跟你去才有意义"。当前 flat regex 只能看词，看不到句子和句子之间的关系。

## 新增内容

### 模板（4个）

| # | 名称 | 结构 | 示例 |
|---|------|------|------|
| 1 | setup-invitation | S1陈述 → S2邀请(含"你") | "鸡鸣寺的樱花开了。你要和我一起去看看吗？" |
| 2 | indirect-exposure | S1无关日常 → S2突然自我暴露 | "今天天气挺好的。其实我昨天晚上梦见你了。" |
| 3 | self-dialogue | S1自问 → S2自答 | "是不是我想太多了？可能是吧。" |
| 4 | 条件依附 | S1美好场景 → S2"没你这就不算数" | "外面的樱花开了。可惜你不在，也没什么意思。" |

### 关键修复

- `_split_sentences()` 从 `re.split(r'[。！？!?\n]+')` → `re.split(r'(?<=[。！？!?\n])')` ——保留标点，否则"是不是？"拆出来变成"是不是"，`?` 丢失后自问模板永久触不到

### 集成点

- `extract()` 末尾调用 `_detect_sentence_structures()` 写入 `signals.sentence_structures`
- `_determine_movement()` 中四种模板全部判为"拉近"——敢绕圈子说话本身是关系近的信号
- `_build_context()` 中句间结构排在僵化之前
- `_identify_key_moment()` 中四个分支分别输出自然语言 key_moment

## 设计边界

- **只读句间形态，不碰语义** ——符合 SKILL.md 原则
- 和词级信号并行，互不覆盖
- 单句不触发（最少两句）
- 每个模板用 `break` 避免同一模板在多句匹配中重复触发
